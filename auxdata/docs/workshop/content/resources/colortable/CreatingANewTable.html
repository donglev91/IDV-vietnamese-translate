[ht::desc In this section we will create a color table to use within the IDV.]
In this section we will create a color table to use within the IDV. This example color table is intended to represent a comfort range of temperatures.
  1. Open up the Color Table Editor from the [ht::menu Tools {Color Table Editor}] menu if it is not already open.
  2. Select the [ht::ct default] color table if it is not already displayed. Select the [ht::menu {Color Tables} Basic default] menu item.
  3. Rename this color table by selecting the [ht::menu File {Save As...}] menu and entering the name [ht::qt Comfort].
  4. You should now have the same looking color table but with a new name displayed. We will create a new category, [ht::qt Workshop] for this color table. Color table categories have no actual semantic import. They are just a simple way to organize a (potentially) long list of color tables.
    • Click in the [ht::command Category] box and type [ht::qt Workshop] and press return.
    • Now save this new color table by selecting the [ht::menu File Save] menu item.
    • Verify that you indeed have a new category by selecting the [ht::menu {Color Tables}] menu.
      • You should see: [ht::cimg images/WorkshopCategory.gif] Note: The label is [ht::qt Comfort (local)] The [ht::qt (local)] denotes that this color table is one that you have created, not a system provided color table.
  5. Now, note the Range values 0.0 - 100.0.
    The range values in the Color Table Editor are there for convenience purposes only. They allow you to see at what areas in the color table certain values would be shown in actual use.
    We are going to set these to an example range for temperature in Celsius.
    • Type in -45 in the first box and press return.
    • Type in 45 in the second box and press return.
    • You should see the new range values displayed below the color box.
  6. Add breakpoints at the 0 value (freezing), 15 (comfortable), and 25 (hot).
    • Right click on the color bar to bring up the [ht::menu {Color Bar Menu}]:
      • [ht::cimg images/AddBreakpointMenu.gif]
    • Select [ht::menu {Add Breakpoint} Here]
    • You should see a color bar with a new breakpoint in the middle:
      • [ht::cimg images/FirstZeroBreakpoint.gif]
    • Click and drag the breakpoint to the 0 value. If you cannot get to exactly 0 then use either the right/left arrow keys or right mouse click on the breakpoint and select the [ht::menu {Change Breakpoint}] menu item and enter the desired value.
    • An alternative method would be to add a breakpoint at a particular value is to select the [ht::menu {Add Breakpoint} {At Data Point}] menu item and enter the value 0.
    • Repeat the above steps for the values 15 and 25.
    • Your color bar should look like:
      • [ht::cimg images/ThreeBreakpoints.gif]
  7. Now, let's come up with an interesting color scheme.
    • Click on the [ht::button Fill Edit Mode] button.
    • Select the center breakpoint (i.e., the [ht::qt 0] value point) by clicking on it.
    • Set the color under this break point to white by using the color chooser.
    • Right click on the center breakpoint and select the [ht::menu {Set Colors} {Interpolate Left}] menu item.
    • How the rest of the Comfort color table is colored is an exercise left to the reader.
  8. Once you are done you should save this color table by selecting the [ht::menu File Save] menu item.
