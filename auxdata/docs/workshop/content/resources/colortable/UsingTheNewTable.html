Now we are going to use the new color table that we created in the previous section.
  1. First, load in some data and an example display from the [ht::menu Displays {Favorite Bundles} {ToolBar} {Workshop} {Color table exercise}] menu.
  2. Note the color table that is used. By default this display uses the [ht::ct Temperature] color table for the [ht::param temperature] field.
  3. Change the color table for this display to the new color table, [ht::ct Comfort] by clicking on the [ht::button Color Table] button to popup the Color Table menu.
  4. Change the range.
    • Note the range that is being used is -90.0 to 45.0. This is the default range for [ht::param temperature] data. However, our [ht::ct Comfort] color table was set up to use -45 to 45 as a range.
    • Change the range through the color table menu by clicking on the "Comfort" color table. You can also click on the legend's color bar or through the display control's [ht::menu Edit {Color Table} {Change Range...}] menu. Enter the values -45, 45 and click the [ht::button OK] button.
  5. You can quickly change the entire color table for this display.
    • Click on the "Comfort" color table to show the popup.
    • Select [ht::command Dimmer].
    • Select [ht::command Brighter].
    • You can also change transparency.
  6. Bring up the [ht::dialog Color Table Editor] by clicking on the color bar in the control window.
    • Changes to this color table are automatically applied to the display unless the [ht::button Auto update] checkbox is unchecked.
    • Let's make our [ht::qt comfort] zone be semi-transparent.
      • Select 80% transparency.
      • Right click on the breakpoint at 15 and select [ht::menu {Set Colors} {Transparency Right}].
      • Click the [ht::button Apply] button if [ht::button Auto update] is turned off.
