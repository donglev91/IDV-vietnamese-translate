The [ht::index {} smed {Layout Model Editor}] Layout Model Editor allows you to view, change and create Layout Models.
  1. First, load in the layout model example from the [ht::menu Toolbar {Workshop} {Station model exercise}] menu.
  2. Open up the [ht::dialog Layout Model Editor] from the [ht::menu Tools {Layout Model Editor}] menu.
  3. You should see something like this:
    4. [ht::cimg Overview.gif]
  4. The [ht::dialog Layout Model Editor] consists of:
    • A drawing canvas.
    • A list of symbols.
      • [ht::command Select] Is an editing command. Puts the editor into select mode.
      • [ht::command Text] Displays text. Used for station identifiers, text results from Jython, etc.
      • [ht::command Value] Displays a numeric value. Lets you specify a display unit and a format.
      • [ht::command Label] Shows a fixed text label.
      • [ht::command Shape] Shows a shape.
      • [ht::command Mapped Shape] Maps a value to a shape.
      • [ht::command Windbarb] Shows wind direction and speed.
      • [ht::command Cloud coverage], [ht::command Present weather], [ht::command Low cloud], etc. These symbols are used to map a value into a predefined range of display symbols.
    • An alignment toolbar.
    • [ht::menu File], [ht::menu Edit], [ht::menu View], [ht::menu {Layout Models}] and [ht::menu Help] menus.
  5. Let's go look at the [ht::filename METAR] layout model.
    • Right click on the symbol in the upper left that represents temperature and select [ht::menu Properties]
    • Change some of the properties (e.g., scale, background color, format) and click [ht::button Apply].
    • The [ht::button Apply] and [ht::button OK] buttons save your changes to the layout model in the editor.
    • The [ht::button Save] button saves the changes plus writes out the layout model and updates the display.
    • Close the dialog and use the [ht::menu File Remove] menu to get rid of your local changes. Re-select the [ht::filename METAR] layout model with the [ht::menu {Layout Models}] menu.
    • Try moving some the position of the symbols around. To apply the change to the display use the [ht::menu File Save] menu item.
    • Look at the [ht::filename Earthquake] layout model. View the properties for the symbol.
  6. Create a new Layout Model, called "Workshop".
    • Select the [ht::menu File New] menu item.
    • In the dialog box enter the name of the new layout model, [ht::qt Workshop].
    • Click the [ht::command OK] button.
  7. Add a symbol.
    • Select the [ht::command Text] symbol in the symbol list.
    • Click in the center of the drawing canvas.
    • In the [ht::dialog Properties] dialog you can select what parameter from the data set is used and other display information.
      • The value in the [ht::command Parameter] field should be [ht::qt IDN], the WMO Station ID.
      • Select Courier in the [ht::menu Font] menu.
      • Select an 11 point font in the [ht::menu Size] menu.
      • Click the [ht::button OK] button.
  8. Position the symbol.
    • Click and drag the symbol to the center of the screen.
    • Note the red dot on the symbol. This is the Alignment point. It determines the point on the bounding box of the symbol that is used to align the symbol with the center point of the display coordinate space.
    • Right click on the symbol to bring up the [ht::menu {Symbol Menu}]
    • Select the [ht::menu {Alignment Point} NW] menu item. This should change the red dot on the symbol to the upper left corner.
    • Right click on the symbol again, but this time select the [ht::menu Center Center] menu item. This aligns the symbol's alignment point to the center of the display space.
  9. Save this layout model with the [ht::menu File Save] menu item.
  10. Close the window by selecting the [ht::menu File Close] menu item.
  11. Back to the [ht::display Point Data Display], choose the [ht::filename Workshop] layout model. Humm, why isn't anything displayed?
