In this section, we will cover how to capture images within the IDV.
  1. If you have nothing displayed, then load in the bundle that we previously created.
    • Select the [ht::menu File Open] menu item. In the [ht::dialog File Dialog] choose [ht::filename workshop.xidv] from your home directory.
  2. Select the [ht::menu View Capture {Image...}] menu item from the map view's menu bar. The [ht::dialog File Dialog] will popup.
  3. The IDV requires that the window you are capturing an image in is not obscured. Move any other windows out of the way.
  4. You can save an image as a jpg, gif or png. [ht::save workshop.jpg]
  5. Let's capture the full window. From the File Dialog enter [ht::filename fullwindow.png] and select the [ht::button Full Window] button on the left.
  6. The IDV treats image files as yet another data source. Load in the newly created images by:
    • [ht::openDataSourceChooser]
    • Go to the [ht::command Files] tab.
    • Make sure you are in your home directory.
    • Select [ht::filename workshop.jpg] and [ht::filename wholewindow.png] with a control-click
  7. Since images are not geo-located we can only display the image in the [ht::display Omni Control]. Create this display.
  8. Now, let's save this image as a Google Earth kmz file.
    • When we capture an image for Google Earth we have to make sure we are using a Lat/Lon projection in an overhead view.
    • The easiest way to do that is to select the [ht::menu Projections {Use Displayed Area}] menu.
    • Select the [ht::menu View Capture {Image}] menu item and enter [ht::filename workshop.kmz].
    • A reminder is shown about the Lat/Lon projection.
    • From a terminal window bring up Google Earth and Open this kmz file.
  9. The IDV can also read in Google Earth kml/kmz files.
    • From the IDV Data Source Chooser select the [ht::filename workshop.kmz] file.
    • From the field selector create an Image display.
